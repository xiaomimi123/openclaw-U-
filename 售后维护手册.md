# OpenClaw U盘版 · 售后维护手册

**适用版本：** v1.1.0 及以上
**更新日期：** 2026-03-22

---

## 一、U 盘版文件结构（必须了解）

```
U 盘根目录/
├── OpenClaw-U盘版.exe     ← 启动器主程序（可更新）
├── openclaw.zip           ← OpenClaw 核心（可更新）
├── runtime/
│   └── node.exe           ← Node.js 运行时（极少更新）
└── openclaw-data/         ← ⚠️ 用户数据，【绝对不能覆盖】
    ├── setup.json         ← 用户配置（AI 服务商、聊天工具）
    └── .openclaw/
        ├── openclaw.json  ← OpenClaw 网关配置
        └── extensions/    ← 用户安装的插件（如微信）
```

**更新规则：**
- ✅ 可以覆盖：`OpenClaw-U盘版.exe`、`openclaw.zip`、`runtime/`
- ❌ 禁止覆盖：`openclaw-data/` 及其所有子目录（用户的 AI 配置、对话记录全在这里）

## 二、用户身份核验

### 用户操作（30 秒内完成）

1. 插入 U 盘，运行 `OpenClaw-U盘版.exe`
2. 启动器底部点击 **「📋 授权信息」** 按钮
3. 弹窗显示：

```
版本号:     v1.0.0
U盘序列号:  A1B2C3D4
平台:       win32 / x64
```

4. 点击 **「一键复制」**，发给客服

### 客服核验

- `U盘序列号` = 购买时登记的授权码，两者一致即可确认身份
- `版本号` 决定发哪个更新包

## 三、更新包制作流程（开发侧）

### 修复 Bug 后的发包步骤

```bash
# 1. 修改 package.json 中的版本号（必须）
#    v1.0.x = 小修复；v1.x.0 = 功能更新

# 2. 打包
npm run build

# 3. 制作更新包（只需发变更的文件）
```

**更新包内容参考：**

| 场景 | 发哪些文件 |
|------|-----------|
| 仅修复启动器 Bug | `OpenClaw-U盘版.exe` |
| 更新 OpenClaw 核心 | `openclaw.zip` |
| 两者都改了 | `OpenClaw-U盘版.exe` + `openclaw.zip` |
| Node.js 版本升级 | `runtime/node.exe` |

**打包命令（生成发给用户的 zip）：**
```powershell
# 在 openclaw-usb 目录执行
$ver = (Get-Content package.json | ConvertFrom-Json).version
Compress-Archive -Path "dist\OpenClaw-U盘版.exe","dist-extra\openclaw.zip" `
  -DestinationPath "releases\OpenClaw-U盘版-更新包-v$ver.zip"
```

## 四、用户收到更新包后的操作步骤

发给用户的说明（可直接复制发微信/邮件）：

---

**【OpenClaw U盘版 · 更新说明】**

您好！以下是针对您的 U 盘版的修复更新包，操作非常简单：

1. **关闭** OpenClaw（如果正在运行，右下角退出）
2. 打开您的 U 盘
3. 将更新包中的文件 **直接复制到 U 盘根目录**，选择"覆盖"
   - ⚠️ 只复制 `.exe` 和 `.zip` 文件，**不要动 `openclaw-data` 文件夹**
4. 重新运行 `OpenClaw-U盘版.exe` 即可

您的所有 AI 配置和对话记录不受任何影响。

## 五、常见问题处理

### 5.1 用户说"打开就闪退"

**排查：**
1. 询问版本号（点授权信息查看）
2. 让用户右键 exe → 属性，确认文件大小是否 > 50MB（太小说明下载不完整）
3. 发新版 `OpenClaw-U盘版.exe` 覆盖

### 5.2 用户说"卡在解压，一直 0%"

**原因：** 旧版缺少 yauzl 依赖（v1.1.0 已修复）

**处理：** 发送 v1.1.0 `OpenClaw-U盘版.exe` 覆盖，同时让用户手动删除：
```
C:\Users\{用户名}\AppData\Local\Temp\openclaw-usb\
```
然后重新运行

### 5.3 用户说"切换 AI 服务商后 API 报错"

**原因：** 旧版切换时 baseUrl 残留（v1.1.0 已修复）

**处理：** 发 v1.1.0 exe + 指导用户点「🔑 修改 API Key」重新选择服务商并输入 Key

### 5.4 用户说"微信扫码后没反应"

**处理步骤：**
1. 确认插件已安装（启动器显示"已安装"）
2. 让用户点「扫码连接」，在弹出的黑色命令行窗口扫码
3. 扫码成功后，在启动器点「⏹ 停止」→「▶ 重新启动」
4. 仍不行：让用户检查微信是否在手机上确认了授权

### 5.5 用户换了新电脑，U 盘无法启动

**原因：** 授权绑定的是 U 盘序列号，跟电脑无关 —— 理论上任何电脑都能用

**排查：** 新电脑是否有 Node.js 要求的运行库？（Windows 7 不支持，Win10+ 均可）

### 5.6 用户 U 盘损坏，数据能否恢复？

**结论：** 无法从服务端恢复，因为所有数据只存在 U 盘上

**预防措施（告知所有用户）：**
- 定期备份 U 盘中的 `openclaw-data/` 文件夹到电脑
- 该文件夹包含全部配置和对话记忆
